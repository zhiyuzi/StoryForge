# Eval: 低质量检测 — 故意缺陷的梗概能否被正确识别

## 类型
评估体系验证（Assessment Validation）

## 优先级
P1

## 目的
验证评估 Agent 能否正确识别低质量内容，不会对明显有缺陷的梗概给出高分。

## 输入
一个故意设计了缺陷的梗概，包含以下问题：
- 核心冲突在第 3 集突然消解，后续无冲突
- 无爽点设计
- 结尾钩子全部缺失
- 付费墙设在第 2 集（太早，免费内容不足以吸引用户）

## 执行步骤

### Step 1：准备低质量梗概
创建 projects/test-low-quality/synopsis/v1.md，内容为一个 10 集复仇短剧梗概，但故意包含上述缺陷。

### Step 2：运行评估
对 v1.md 运行叙事评审 Agent 和逻辑审计 Agent 的评估。

### Step 3：验证评估结果
检查评估报告：
- 冲突处理维度应 ≤ 2/5（冲突在第 3 集消解）
- 爽点密度应 ≤ 2/5（无爽点设计）
- 钩子强度应 ≤ 2/5（结尾钩子缺失）
- 付费点优化应 ≤ 2/5（付费墙位置不当）
- 综合判定应为"需修改"

## 评估方法
- 分数验证：检查各维度分数是否正确反映了缺陷
- 分析验证：检查评估报告是否准确指出了具体缺陷位置

## Pass 标准
- 冲突处理 ≤ 2/5
- 爽点密度 ≤ 2/5
- 钩子强度 ≤ 2/5
- 付费点优化 ≤ 2/5
- 评估报告中明确指出了上述 4 个缺陷
